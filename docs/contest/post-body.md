## 一、创意名称与定位

**创意名称：** Homework Buddy · 智听

**一句话定位：** 基于微信小程序 + 微信云开发 + 火山 OCR + 豆包大模型的 AI 自动化听写助手，让家长彻底告别“人工报词机”，让孩子获得标准发音、稳定节奏、可追踪进步的听写训练。

**为什么叫 Homework Buddy：** 它不是冷冰冰的工具，而是孩子写作业时的“伙伴”——会报词、会批改、会整理错题，也会帮老师布置全班作业。Buddy 一词强调陪伴感与可用性。

**为什么中文叫“智听”：** “智”对应 AI 智能识别、智能抽题、智能批改；“听”直指听写训练核心场景。英文名与中文名组合为统一品牌 `Homework Buddy · 智听`。

---

## 二、核心创意：把「报听写」这件高频小事，彻底自动化

每个有学龄孩子的家庭，几乎每天晚上都会上演同一幕：

1. 家长放下手机/关掉电脑，拿出课本；
2. 逐字朗读 10-20 个单词或词语；
3. 控制节奏、重复、等待；
4. 批改、纠错、手抄错题本。

这个过程每天 20-30 分钟，一年累计超过 **80 小时**。更关键的是，家长发音不准、节奏不稳、情绪疲惫，都会直接影响孩子的学习效果。

**Homework Buddy 用 AI 把上述五步压缩成三步：**

```
拍照/上传课本  →  AI 识别并结构化  →  开始听写 + 自动批改
```

家长只需按一次快门，剩下的报词、计时、批改、归档全部由系统完成。

## 三、目标用户与使用场景

### 核心用户
- **小学生家长**：被报听写占据时间，希望解脱；担心自己发音不准误导孩子。
- **小学生**：需要标准发音、稳定节奏、能反复练习错题。
- **小学语文教师/英语老师**：需要给全班布置统一听写任务，并追踪每位学生的正确率。

### 典型场景
| 场景 | 传统方式 | Homework Buddy 方式 |
|------|---------|---------------------|
| 每日家庭听写 | 家长全程陪同 20-30 分钟 | 孩子独立 5-8 分钟完成 |
| 周末单元复习 | 手动整理整单元词汇 | 勾选单元一键生成复习卷 |
| 考前突击 | 家长反复报词，效率低 | 错题本一键重测，精准补漏 |
| 班级统一作业 | 老师在黑板抄题、家长配合 | 班级码共享词库，一键下发 |

## 四、解决的五大痛点

1. **家长精力被绑架**
   - 每天晚上必须空出完整时间段陪同，无法处理其他事务。
   - *Buddy 方案：* 孩子独立操作，家长只需查看结果。

2. **发音不标准误导孩子**
   - 家长英语口音、声调不准，孩子反而记住错误发音。
   - *Buddy 方案：* 微信同声传译插件 / 大厂 TTS 标准发音，覆盖率 100%。

3. **节奏全凭感觉**
   - 报太快写不完，报太慢拖时间，无法固定训练强度。
   - *Buddy 方案：* 可配置词间等待时间（3s / 5s / 8s），节奏稳定可控。

4. **批改整理繁琐易错**
   - 人工对答案耗时，错题难以系统归档。
   - *Buddy 方案：* 提交后秒级自动批改，错误高亮，自动入错题本。

5. **词库维护困难**
   - 课本内容要手动录入，工作量大且格式混乱。
   - *Buddy 方案：* 拍照 → OCR → 大模型清洗，3 步生成标准词库；也支持手动加词。

## 五、六大核心亮点（相比传统方案的提升点）

### 1. 拍照识字：3 秒建词库
- 火山引擎 OCR 识别课本照片，支持印刷体、手写标注、倾斜拍摄。
- 豆包大模型自动过滤页码、页眉、无关符号，输出结构化词库。
- **提升：** 从手动录入 30 分钟/单元 → 拍照 3 秒完成。

### 2. AI 结构化清洗：从乱文本到标准 JSON
- 大模型 Prompt 工程，输出字段：`word` / `pinyin` / `meaning` / `partOfSpeech` / `phonetic` / `unitId` / `lesson`。
- 同单元自动去重，避免重复录入。
- **提升：** OCR 原始文本杂乱无章 → AI 直接输出可入库结构化数据。

### 3. 智能抽题引擎：防作弊随机算法
- 支持按单元、课次筛选。
- 支持 Min-Max 数量区间（如 10-15 个），每次随机决定题目数。
- 采用 Fisher-Yates 洗牌算法打乱顺序，避免学生背顺序。
- **提升：** 从固定顺序 → 每次题目数与顺序都不同。

### 4. 标准语音播报：节奏可控
- 微信同声传译插件 TTS，或标准发音服务。
- 可设置词间等待时间，支持重复播放。
- **提升：** 从家长人工朗读 → 标准发音 + 稳定节奏。

### 5. 自动批改与错题本：闭环学习
- 作答后秒级对比标准答案，正确率实时显示。
- 错误单词红笔高亮，自动写入 `userLogs.wrongWords`。
- 支持“一键重测错题”，针对性补漏。
- **提升：** 从人工批改 → 秒级反馈 + 长期追踪。

### 6. 班级共享：从家庭工具到教学工具（规划中）
- 老师生成班级码，全班共享同一套词库。
- 老师可查看班级整体正确率、高频错题。
- **提升：** 从单一家庭使用 → 家校协同的教学基础设施。

## 六、技术栈与架构

### 前端
- **原生微信小程序**（WXML / WXSS / JS）
- 不使用 Uni-app/Taro，保证运行性能与微信 API 原生体验。

### 后端
- **微信云开发 CloudBase**
  - 云函数：Node.js 18
  - 云数据库：NoSQL / JSON 风格
  - 云存储：存放语音、图片、HTML 产物

### AI / OCR
- **火山引擎 OCR**：图片转文字
- **豆包大模型**（OpenAI SDK 兼容模式）：文本结构化清洗
- API Key 全部存放在云函数环境变量，前端零密钥暴露。

### 语音
- **微信同声传译插件 TTS**：标准发音、低延迟

### 核心数据模型
- `words`：单词表
- `units`：单元/课次表
- `userLogs`：听写记录与错题本

## 七、社会价值与意义

### 对家庭的效率提升
- 每个家庭每天节省 25 分钟，一年约 200 个听写日，累计释放 **80+ 小时**。
- 家长从“报词机”回归“陪伴者”，亲子时间质量提升。

### 对教育公平的促进
- 发音薄弱、教育资源不足的家庭，也能获得与名校课堂同等标准的语音输入。
- 拍照识字 + AI 清洗降低词库建设门槛，缩小数字鸿沟。

### 对教学管理的赋能
- 老师可基于错题数据调整教学重点，实现数据驱动的个性化教学。

## 八、项目现状

### 已落地功能
- 数据库 Schema 设计：`words` / `units` / `userLogs`
- 核心云函数：
  - `getDictationList`：随机抽题引擎（单元/课次筛选、Min-Max、Fisher-Yates 洗牌）
  - `addWord`：手动录入，同单元去重
  - `parseOcr`：火山 OCR + 豆包 LLM 结构化清洗，去重入库
  - `login`：获取用户 OPENID
- 小程序页面：
  - 首页：学科/单元/模式/数量/间隔配置，支持拍照导入、手动加词
  - 听写页：语音播报、答案输入、下一题、自动间隔
  - 结果页：正确率、错题高亮、保存记录、一键重测
  - 拍照页：上传云存储、OCR 识别、结果展示
  - 个人中心：听写记录与错题本
- 工具封装：`db.js` 数据库操作、`aiClient.js` AI 调用封装、`tts.js` 语音播报封装
- 测试：Jest 单元/集成测试覆盖 parseOcr helper、抽题引擎、TTS 语言解析
